ResourceEvent

Swift

public enum ResourceEvent: String

The possible causes of a call to ResourceObserver.resourceChanged(_:event:).

  • SeeAlso: Resource.load()
  • ObserverAdded

    Immediately sent to a new observer when it first starts observering a resource. This event allows you to gather all of your “update UI from resource state” code in one place, and have that code be called both when the UI first appears and when the resource state changes.

    Note that this is sent only to the newly attached observer, not all observers.

    case ObserverAdded
  • Requested

    A load request for this resource started. Resource.loading is now true.

  • RequestCancelled

    The request in progress was cancelled before it finished.

    The resource’s latestData property has been updated.

  • NotModified

    The request in progress succeeded, but did not result in a change to the resource’s latestData (except the timestamp). Note that you may still need to update the UI, because if latestError was present before, it is now nil.
