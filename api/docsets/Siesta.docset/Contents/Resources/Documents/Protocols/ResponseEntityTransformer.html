ResponseEntityTransformer

Swift

public protocol ResponseEntityTransformer: ResponseTransformer

A utility flavor of ResponseTransformer that deals only with the response entity (whether success or failure), and does not touch the surrounding error metadata (if any).

To use, implement this protocol and override the processEntity(_:) method.

  • processEntity(_:) Default implementation

    Implementations will typically override this method.

    Return .Success if the parsing succeeded, .Failure if it failed. See the default implementation of processError(_:) for information about what happens to this method’s return value when the underlying response is a failure.

    Default Implementation

    Returns success with the entity unchanged.

    Declaration

    Swift

    func processEntity(entity: Entity) -> Response
  • processError(_:) Default implementation

    Implementations typically do not override this method, but they can if they wish to apply special processing to errors. For example, an implementation might want to leave errors untouched, regardless of content type, and only apply processing on success.

    Default Implementation

    Attempt to process error’s `content` entity using `processEntity(_:)`. If the processing succeeded, replace the

    content of the error. If there was a processing error, log it but preserve the original error.
