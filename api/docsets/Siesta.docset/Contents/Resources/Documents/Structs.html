  • ResourceError

    Information about a failed resource request.

    Siesta can encounter errors from many possible sources, including:

    • client-side parse issues,
    • network connectivity problems,
    • transport layer issues (e.g. certificate problems),
    • server errors (404, 500, etc.), and
    • client-side parsing and data validation failures.

    ResourceError presents all these errors in a uniform structure. Several properties preserve diagnostic information, which you can use to intercept specific known errors, but these diagnostic properties are all optional. The one ironclad guarantee that ResourceError makes is the presence of a userMessage.
