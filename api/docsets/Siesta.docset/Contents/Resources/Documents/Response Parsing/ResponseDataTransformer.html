  • processData(_:)

    Subclasses will typically override this method. Default is to leave data unchanged.

    Note that overrides can turn a success into an error, e.g. if there is a parse error.

    Default behavior: attempt to process error response bodies just like success bodies, but if there is a transformation error, only log it and preserve the original error.
