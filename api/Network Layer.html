  • RequestNetworking

    Network handling for a single request. Created by a NetworkingProvider. Implementations have three responsibilities:

    • start the request when start(_:) is called,
    • call the closure passed to start(_:) is called when the request is complete, and
    • optionally support cancelling requests in progress.

  • AlamofireProvider

    Uses Alamofire for networking. This is Siesta’s default networking provider.

    You can create instances of this class with a custom Alamofire.Manager (or, for convenience, a custom NSURLSessionConfiguration) in order to control caching, certificate validation rules, etc. For example, here is a Service that will not cache anything and will not use the cell network:

    class MyAPI: Service {
    init() {
        let configuration = NSURLSessionConfiguration.ephemeralSessionConfiguration()
        configuration.allowsCellularAccess = false
        super.init(
            base: "http://foo.bar/vi",
            networkingProvider: AlamofireProvider(configuration: configuration))
    }
}
